# GameRound - Execution Entity

The `GameRound` class captures the complete lifecycle of a single unit of gameplay.

## 🎯 Purpose
To serve as the "Truth" of what happened. If a player disputes a result ("I spun 7 but you paid me 0!"), this record contains the inputs and the exact RNG result to prove the outcome.

## 🏗️ Property Breakdown

| Property | Type | Description |
| :--- | :--- | :--- |
| **`Id`** | `Guid` | Unique ID for this specific hand/spin. |
| **`GameSessionId`** | `Guid` | Links to the user's active session. |
| **`RoundNumber`** | `int` | An incremental counter (1, 2, 3...) relative to the session. Used with the **Seed** to generate deterministic RNG. |
| **`InputData`** | `string` | **Snapshot of Player Choice**. Stores what the user asked for (e.g., "Hit" actions in Blackjack, or the specific bet placements in Roulette). |
| **`RandomResult`** | `string` | **Snapshot of Server Logic**. Stores the raw result generated by the RNG engine (e.g., `{"Cards":["Ah","Ks"]}` or `{"WinningNumber": 17}`). |
| **`TotalBetAmount`** | `decimal` | The total cash wagered in this round. |
| **`TotalWinAmount`** | `decimal` | The total cash paid out. If 0, the player lost. |
| **`ExecutedAt`** | `DateTime` | Precision timestamp of the round completion. |

## 🔐 Verification
By combining `GameSession.Seed` + `GameRound.RoundNumber`, an auditor can mathematically recreate the `RandomResult` to prove the casino didn't cheat.
